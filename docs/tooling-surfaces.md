# Tooling Surfaces

Marrow tools render compiler, catalog, runtime, and store facts. They do not
define another database model, and a debug surface does not become a production
API by being documented.

A shared tooling facts layer owns typed data-path resolution, path rendering,
bounded previews, integrity findings, catalog/snapshot metadata, and cursor
validation. CLI commands and backup/restore are renderers over those facts.

## Surfaces

| Surface | Support level | Boundary |
|---|---|---|
| `marrow data roots` / `stats` | Operator/admin inspection. | Exact scans under one stable snapshot are allowed for admin commands; not a production preview API. |
| `marrow data dump` | Operator/admin inspection. | Walks a full stable snapshot by explicit operator request and may expose canonical payload bytes; not a backup/restore format, sync format, production preview, or production data API. |
| `marrow data get` | Operator/admin point inspection. | Presence states are typed facts; raw payload bytes are diagnostic output. |
| `marrow data integrity` | Read-only data-integrity tooling. | Reports decode, key-type, dangling-reference, incomplete, and orphan findings with typed fields; does not bless invalid managed data. |
| `run --trace` / `test --trace` | Debug execution rendering. | Observes runtime statement/write facts over checked source spans; does not change run semantics; not a stable external API. |
| `run --dry-run` | Checked write preview for one operator-run entry. | Previews that run's managed writes against an isolated run store; use `evolve preview` for evolution. |
| `--maintenance` | Explicit operator capability. | For modeled repair/evolution code, not raw store mutation; cannot be injected by project config or a default entry. |
| `marrow backup` / `restore` | Production typed backup/restore. | Carries source digest, accepted catalog epoch, engine descriptor, and typed cells; restore writes into an empty store unless `--replace --count N` matches the live record count, rebuilds generated indexes, rejects orphaned managed cells before commit, and rolls back on `restore.data_invalid`. |
| Checked entry descriptors / `marrow-run::EntryInvocation` | Unstable linked-Rust entry invocation. | `marrow-check` renders `entry.invoke.v1` descriptors from checked public entry signatures, parameter shapes, return shape, accepted catalog identities, and return presence; `marrow-run` admits typed invocation values against the current descriptor identity and executes them. It is not a JSON request profile, HTTP route, generated-client contract, or stable public Rust API. |
| Checked `SurfaceFact` / operation facts / operation descriptors | Unstable checked application-surface facts. | Resolves declared surfaces to checker-valid store, member, index, read kind, footprint, projection identities, read-operation aliases, create fields, sparse update fields, delete declarations, public action functions, public read-only computed-read functions, and accepted-catalog operation descriptors/tags for stable surfaces. These facts are transport-neutral compiler facts; they are not HTTP routes, TypeScript names, raw saved paths, or an admin data-access API. Their typed catalog status reports whether stable descriptor export is blocked by a pending catalog proposal or missing accepted catalog IDs, including durable IDs needed by generated write fields and action/computed-read parameter or return types. |
| `marrow-run::ProjectSurfaceReadSession` | Unstable linked-Rust surface read session. | Checks a project, opens the configured native store read-only, requires an already accepted and stamped durable store, fences drift without auto-apply, exposes admitted surface read and computed-read operations by stable operation tag, and reports an admitted serve boundary with read-only mode, data-view source/store/watch facts, and no exposed process-control endpoint. It is not HTTP routing, process lifetime, generated clients, create/update/delete/action execution, opaque cursor tokens, UID minting, baseline freeze, maintenance, restore, recovery, debugger attach, or a stable public Rust API. |
| `marrow-run::ProjectSurfaceSession` | Unstable linked-Rust surface read/write session. | Checks a project, opens an existing configured native store writable, requires accepted catalog identity plus store UID and commit metadata, fences drift without hidden repair, exposes admitted surface reads, computed reads, creates, sparse updates, deletes, and actions by stable operation tag without exposing the store handle, and reports an admitted serve boundary with write mode, data-view source/store/watch facts, and no exposed process-control endpoint. It is a single-owner, sequential session; while it is open, the native writer lock makes it the owning process/session and excludes another writer or read-only inspection. It is not HTTP routing, process lifetime, generated clients, opaque cursor tokens, UID minting, baseline freeze, store creation, auto-apply, maintenance, restore, recovery, debugger attach, or a stable public Rust API. |
| `marrow-json::surface` DTOs | Checked application surface JSON. | Renders serialized surface ABI descriptors and descriptor-derived route manifests for `marrow check --format json|jsonl`, including checked read/computed-read/action aliases as labels; validates route manifests against matching ABI operation catalogs/profiles; renders the typed TypeScript surface client and the explicit cursor-token TypeScript profile; owns the `surface.cursor_token.v1` codec over existing typed cursor DTOs; decodes checked read request parameters through admitted runtime reads, including scalar range bounds for ranged index pages; executes read and computed-read DTOs over `ProjectSurfaceReadSession`; decodes create/update/delete request DTOs through admitted runtime generated write handles over caller-supplied stores or `ProjectSurfaceSession`; decodes action arguments through `entry.invoke.v1` and executes actions over `ProjectSurfaceSession`; dispatches `surface.operation.v1` request envelopes by stable operation tag through `ProjectSurfaceReadSession` or `ProjectSurfaceSession`; and serializes already-executed surface records, pages, typed values, catalog identities, commit-bound context-aware typed cursor boundaries, accepted-catalog action and computed-read values, created/updated/deleted/action/computed-read results, and sanitized code/message error envelopes. Ranged index page reads share the single profile and route under `/surface/v1/read/<operation-tag>`. The route manifest names JSON `POST` paths over operation tags; it is not a listener or process-lifetime policy. The generated TypeScript decodes response values against the descriptor as a convenience projection and fails loud on a malformed field, but does not become an authority boundary. Computed reads always use a zero-capability host, and host-effecting computed reads are rejected by the checker. The default project operation helper also gives actions a zero-capability host; callers that need action host capabilities use the explicit-host helper. It is not HTTP serving or a raw saved-data access API. |
| `marrow-json::serve` DTOs | Admitted serve-boundary JSON. | Renders `SurfaceServeBoundary` from an already admitted read-only surface session, write surface session, or run session as serve mode, admitted data-view boundary, and process-control status. `processControl.kind = "not_exposed"` means the linked-Rust boundary is not a served-process identity, debugger attach target, listener, port, process ID, or control channel. |
| `marrow client typescript` | Local generated-client rendering. | Checks a project against the committed catalog artifact without opening or mutating the saved-data store, then renders a self-contained typed TypeScript client over `surface.operation.v1` and `surface.route.v1` exporting a `createClient` factory, branded id constructors, typed records, a `MarrowSurfaceError` class with a closed `SurfaceErrorCode` union, and an `invokeRaw` escape hatch. `--cursor-token` renders the remote cursor-token profile with branded string page cursors and a distinct client digest; the default and declared client path remain the typed cursor profile. With `--out <path>` it writes the client there, resolving a relative path against the current directory and reporting the resolved path; without `--out` the default profile refreshes the declared `marrow.json` `client` path write-if-changed when one is set, otherwise it prints to stdout. The default client is also the project's declared compile-fresh output: when `marrow.json` sets `client`, `run`, `serve` startup, and `evolve apply` rewrite that path write-if-changed on a surface-ABI change, so this command refreshes the same declared path and is the ad-hoc escape hatch for an out-of-tree copy. A committed `marrow.lock` behind the current source (the `check.stale_lock` condition) is reported on stderr before generation, naming the run that re-projects it. It flattens to surface-name keys, sanitizes render names, keeps operation tags and catalog IDs as private constants, decodes Marrow `int` to `bigint` and rejects unsafe `number` inputs, and decodes each response value against the descriptor as a convenience projection that fails loud on a malformed field. It is not serving, authentication, response payload proof, or a bypass around server/runtime validation. |
| `marrow serve` | Application-surface serving. | Foreground HTTP server over descriptor-derived `/surface/v1/{read\|create\|update\|delete\|action}/<operation-tag>` routes and `surface.operation.v1` request envelopes, including ranged index page reads under `/surface/v1/read/<operation-tag>`. Default mode binds loopback, opens `ProjectSurfaceReadSession`, and serves read routes, including computed reads and ranged pages; `--write` opens `ProjectSurfaceSession` and also serves create/update/delete/action routes under the native single-owner writer lock. It serves JSON `POST` only, enforces exact route/profile/body operation agreement, can opt into CORS for one exact loopback browser origin with `--cors-origin`, returns sanitized `surface.*` error envelopes, processes at most one request per connection, and closes every response. The remote profile requires Bearer auth, can opt into exact remote CORS, and can wrap page cursors in opaque `surface.cursor_token.v1` strings while keeping non-page operations unchanged. With `--watch` it polls the source-root `.mw` files for a modification-time change between connections and, on one, re-checks the project, rewrites the declared client write-if-changed, and keeps serving the fresh surface; the poll is dependency-free and adds no watcher crate. It is not generated clients, store creation, catalog repair, maintenance, or a raw saved-data API. |

## Boundaries

These hold across every surface:

- Raw saved paths are not a production API. There is no public raw saved-path
  encoder/decoder compatibility surface; tools work from parsed source paths
  and checked/catalog facts.
- Raw payload bytes are debug/admin output only. They may appear in
  machine-readable inspection output as base64; they are not a production
  payload contract.
- Unbounded scans are allowed only for explicit operator/admin commands;
  production previews must be bounded or paged.
- The only local application APIs are the checked-surface HTTP profile exposed
  by `marrow serve` and the generated TypeScript client (`createClient`). The
  client is a declared compile-fresh output: it carries `marrow-surface-digest`
  and `marrow-client-digest` headers, `run`/`serve`/`evolve apply` keep its
  declared path fresh write-if-changed, and `marrow check --locked` guarantees
  CI fails on a client the surface has outrun. Remote and cursor-token profiles
  are generated from the same shared checked facts, not promoted from raw bytes,
  raw paths, or an ad hoc saved-data access language.
