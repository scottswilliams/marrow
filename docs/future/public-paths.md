# Public paths

This page is later future direction. Public paths are not required for the local
v0.1 beta.

A compiler-recognized publication model could project selected exported
functions or semantic addresses into stable typed URI representations. The
compiler could then check key parsing/formatting, route collisions, detached
wire shapes, and whether an invocation grant is narrower than the callable's
verified demand.

Publication must remain distinct from storage and authority:

- a private durable place is not automatically public;
- a source rename does not silently change a public path;
- exact read authority does not imply collection traversal;
- publishing a route grants no permission; and
- physical key bytes never become URLs.

An HTTP or routing library could consume verified publication metadata. Public
transport, error confidentiality, timing, cache invalidation, and principal
policy require their own threat and operational models.
